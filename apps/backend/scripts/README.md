# JARV Backend Scripts

Utility scripts for database management, migrations, and system administration.

## Database Management

### db.py
Main database management script with multiple commands:

```bash
# Run all pending migrations
python scripts/db.py migrate

# Rollback last migration
python scripts/db.py rollback

# Show current migration version
python scripts/db.py current

# Show migration history
python scripts/db.py history

# Reset database (DEV ONLY - requires confirmation)
python scripts/db.py reset

# Seed database with test data (DEV ONLY)
python scripts/db.py seed
```

### generate_migration.py
Generate Alembic migrations using autogenerate:

```bash
# Generate migration with default message
python scripts/generate_migration.py

# Generate migration with custom message
python scripts/generate_migration.py --message "add new models"
python scripts/generate_migration.py -m "add indexes"
```

**Note:** Autogenerate detects:
- New tables
- New columns
- Index changes
- Foreign key changes

But does NOT detect:
- Table/column renames (appears as drop + add)
- Changes to column types that are compatible
- Changes to default values in some cases

Always review generated migrations before applying!

## User Management

### setup_admin.py
Create admin users for authentication:

```bash
# Interactive mode (prompts for password)
python scripts/setup_admin.py

# With command line arguments
python scripts/setup_admin.py --username admin --password SecurePassword123!

# Custom username
python scripts/setup_admin.py --username richard
```

## Worker Management

### test_worker.py
Test Celery worker queue:

```bash
# Run all worker tests
python scripts/test_worker.py

# Tests include:
# - Health check task
# - Synchronous task
# - Asynchronous task
```

### worker.sh
Start Celery worker:

```bash
bash scripts/worker.sh
```

Configuration:
- Log level: info
- Concurrency: 4 workers
- Max tasks per child: 1000
- Time limit: 300 seconds
- Soft time limit: 240 seconds

### scheduler.sh
Start Celery beat scheduler:

```bash
bash scripts/scheduler.sh
```

## Development Workflow

### Initial Setup
```bash
# 1. Install dependencies
poetry install

# 2. Set up environment
cp .env.example .env
# Edit .env with your settings

# 3. Run initial migration
python scripts/db.py migrate

# 4. Create admin user
python scripts/setup_admin.py

# 5. Test worker (optional)
python scripts/test_worker.py
```

### Adding New Models
```bash
# 1. Create/modify models in app/models/

# 2. Generate migration
python scripts/generate_migration.py -m "add new models"

# 3. Review migration in alembic/versions/

# 4. Apply migration
python scripts/db.py migrate

# 5. Verify
python scripts/db.py current
```

### Database Reset (Development)
```bash
# WARNING: This deletes all data!
python scripts/db.py reset

# Then re-apply migrations
python scripts/db.py migrate

# And recreate admin user
python scripts/setup_admin.py
```

## Tips

### Migration Best Practices
1. Always review autogenerated migrations
2. Test migrations in development first
3. Backup production database before migrating
4. Use descriptive migration messages
5. Keep migrations small and focused

### Worker Best Practices
1. Monitor worker health with health_check_task
2. Set appropriate time limits for tasks
3. Use retry logic for transient failures
4. Log extensively for debugging
5. Monitor token usage and costs

### Security Best Practices
1. Never commit .env files
2. Use strong passwords (min 8 chars, mixed case, numbers)
3. Rotate credentials regularly
4. Use different credentials for each environment
5. Enable 2FA where possible

## Troubleshooting

### Migration Issues
```bash
# Check current migration version
python scripts/db.py current

# View migration history
python scripts/db.py history

# If migrations are out of sync
python scripts/db.py rollback
python scripts/db.py migrate
```

### Worker Issues
```bash
# Check if Redis is running
redis-cli ping

# Check if worker can connect to Redis
python scripts/test_worker.py

# View worker logs
# (logs appear in terminal where worker was started)
```

### Database Connection Issues
```bash
# Verify DATABASE_URL in .env
# Check PostgreSQL is running
pg_isready

# Test connection
python -c "from app.core.database import engine; import asyncio; asyncio.run(engine.connect())"
```
