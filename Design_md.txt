- Split large work into smaller bounded tasks
- Spawn temporary sub-agents from Lead Agent templates
- Scope each sub-agent to one workspace
- Scope each sub-agent to one task or task batch
- Prevent authority escalation
- Run sub-agents in parallel through the worker system
- Track token cost
- Track runtime
- Track tool usage
- Collect output
- Verify output
- Dissolve sub-agents after completion, failure, or timeout
- Report swarm activity to Live Operations Feed

Sub-agent rules:

- Sub-agent inherits Lead Agent role template
- Sub-agent inherits Lead Agent tool access
- Sub-agent inherits Lead Agent workspace scope
- Sub-agent authority cannot exceed Lead Agent authority level
- Sub-agent is scoped to one workspace
- Sub-agent is scoped to one task or one task batch
- Sub-agent reports output to the spawning Lead Agent
- Sub-agent must not report directly to Richard unless routed through Orchestrator
- Sub-agent dissolves on task completion
- Sub-agent dissolves on failure
- Sub-agent dissolves on timeout
- Sub-agent actions are logged to audit trail
- Sub-agent audit logs must include parent agent reference
- Sub-agent count is configurable globally
- Sub-agent count is configurable per workspace
- Sub-agent cannot access secrets unless the Lead Agent can access them and the task requires it
- Sub-agent cannot spawn further sub-agents unless explicitly configured
- Sub-agent cannot increase authority
- Sub-agent cannot override workspace rules
- Sub-agent cannot override hard boundaries
- Sub-agent cannot disable logging
- Sub-agent cannot disable verification
